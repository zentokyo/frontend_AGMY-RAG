# frontend_AGMY-RAG
# Требования к веб-приложению админ-панели RAG-ассистента

## Общие требования:
Веб-приложение должно иметь понятный административный интерфейс, адаптивную верстку и обеспечивать удобное управление контентом RAG-ассистента только для авторизованных администраторов.  
Поддержка загрузки документов форматов PDF, TXT, DOCX.  
Поддержка минимально необходимых версий современных браузеров (Chrome >= 90, Firefox >= 88, Safari >= 14, Edge >= 90).  

## Функциональные требования:
**Авторизация**  
Администратор должен иметь возможность входа в систему по email (или username) + пароль.  
Сессионная авторизация через JWT (access/refresh токены).  
Все административные операции доступны только после авторизации с проверкой валидности токена на сервере.  

**Управление базой знаний (Knowledge Base)**  
Администратор должен видеть список загруженных документов в виде таблицы.  
Каждый документ отображается с оригинальным именем файла, датой загрузки, размером (если доступно).  
Возможность загрузки новых документов и удаления существующих.  

**Управление базой вопросов (Q&A)**  
Администратор должен иметь полный CRUD для сущности «вопрос».  
Каждый вопрос содержит поля: `question_text`, `reference_answer`, опционально `tags/category`, `is_active`.  
Таблица вопросов с пагинацией для удобного просмотра и редактирования.  

**Пользовательский интерфейс**  
Приложение должно иметь базовую структуру страниц: Login, База знаний, База вопросов, (опционально) Настройки.  
Верстка на основе макетов из Figma с допустимыми отклонениями (не pixel-perfect).  

**Backend и API**  
REST API для авторизации, CRUD вопросов, загрузки/списка/удаления документов.  
Технологический стек: Frontend — React + Tailwind CSS, Backend — Node.js (Express/NestJS), База данных — PostgreSQL.  

## Дополнительные функциональные требования (необязательно):
Расширенные возможности фильтрации и поиска по документам и вопросам.  
Дашборд с базовой статистикой (количество вопросов, документов, активных записей).  
Экспорт/импорт данных в JSON/CSV формате для резервного копирования.
