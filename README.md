# frontend_AGMY-RAG

## Общее описание

Разрабатывается закрытая веб‑панель только для администраторов для управления контентом RAG‑ассистента (пользовательский интерфейс — Telegram‑бот, его не разрабатываем).   
Задачи панели: управление базой знаний (загрузка/удаление материалов) и управление базой вопросов (CRUD «вопрос–эталонный ответ»). 

## Технологический стек

- Frontend: React + Tailwind CSS. 
- Backend: Node.js (Express/NestJS на выбор). 
- Дизайн: Figma. 
- База данных: только PostgreSQL (все сущности проекта хранятся в ней).

## Функциональные требования

### Авторизация

- Вход по email (или username) + пароль.
- Сессионная авторизация через JWT (access/refresh). 
- Все операции администрирования должны быть доступны только после авторизации (сервер должен проверять валидность токена на административных запросах; иначе защиту можно обойти прямым вызовом API, даже если фронт “прячет” кнопки).

### База знаний (Knowledge Base)

- Загрузка документов (например: PDF, TXT, DOCX) в систему.
- Список загруженных документов (таблица): имя/оригинальное имя, дата загрузки, размер (если доступно).
- Удаление документа.
- Без требований к статусам обработки (queued/processing/ready/failed) — пункт удалён.
- Без требований к поиску/фильтрации по названию — пункт удалён.

### База вопросов (Q&A)

- CRUD для сущности «вопрос».
- Поля вопроса: `question_text`, `reference_answer`, опционально `tags/category`, `is_active`. 
- Таблица вопросов с пагинацией.

## UI/UX и верстка

- Верстка на основе макетов из Figma, допускаются разумные отклонения (не требуется pixel‑perfect).
- Базовая структура страниц: Login, База знаний, База вопросов, (опционально) Настройки.

## Backend/API и данные (PostgreSQL)

- REST API для: логина/рефреша токена, CRUD вопросов, загрузки/получения списка/удаления документов.
- Пункты удалены из требований: интеграция с пайплайном обработки документов (парсинг/чанки/эмбеддинги/сохранение в векторную базу) и централизованное логирование действий администратора.
