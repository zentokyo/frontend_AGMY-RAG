# Документация логики сборки проекта

Этот каталог содержит служебные скрипты для сборки и развертывания проекта.

## Скрипт пересборки `rebuild.mjs`

Скрипт `rebuild.mjs` предназначен для полной автоматической пересборки проекта одной командой. Он является полностью кроссплатформенным (работает на Windows, macOS и Linux).

### Как запустить:

Вы можете запустить скрипт напрямую через Node.js:
```bash
node scripts/rebuild.mjs
```

Или используя ярлык в npm-скриптах из корня проекта:
```bash
npm run rebuild
```

### Логика работы:

1. **Сборка Frontend**: Запускается сборка административной панели (`npm run build`). Сборка генерирует оптимизированные HTML/JS/CSS файлы в каталоге `apps/admin-frontend/dist`.
2. **Сборка Backend в Docker**: Запускается пересборка Docker-контейнеров backend-сервисов:
   ```bash
   docker compose --profile dev up -d --build assistant_backend assistant_ingest_worker
   ```
   В процессе этой сборки обновленная статика админ-панели копируется внутрь образа backend для раздачи через FastAPI.

---

## ⚠️ Важное правило обновления

Этот файл `README.md` и файл `rebuild.mjs` **обязаны меняться синхронно** при любых изменениях в логике сборки проекта.

Если вы:
* Добавили новые шаги компиляции;
* Изменили структуру контейнеров или добавили новые сервисы в `docker-compose.yml`, которые требуют пересборки при изменении кода;
* Переименовали профили запуска (например, `--profile dev`);
* Изменили пути к дистрибутивам;

Вы **обязаны** внести соответствующие изменения в `scripts/rebuild.mjs` и отразить это в настоящем `README.md`.
