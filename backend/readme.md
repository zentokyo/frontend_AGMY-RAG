## Описание эндпойнтов

***

### Themes

**GET /themes/**

Получить список теоретических тем

**POST /themes/**

Создать новую теоретическую тему

**GET /themes/{theme_id}/**

Получить тему по id

**GET /themes/{theme_id}/file/**

Получить файл с теорией по теме

### Exams Themes

**GET /exams/themes/**

Получить список экзаменационных тем

**POST /exams/themes/**

Добавить новую экзаменационную тему

**GET /exams/themes/{exam_theme_id}/**

Получить экзаменационную тему по id

### Questions

**GET /questions/**

Получить список имеющихся вопросов

**POST /questions/**

Добавить новый вопрос в систему

**POST /questions/bulk/**

Массовое добавление новых вопросов в систему

**GET /questions/{question_id}/**

Получить вопрос по его id

### Exam

**POST /exams/**

Создать новую экзаменационную сессию

**GET /exams/{exam_id}/**

Получить экзаменационную сессию по id

**GET /exams/{exam_id}/questions/**

Получить все вопросы по экзаменационной сессии

**GET /exams/users/{user_id}/**

Получить список экзаменационных сессий по id пользователя

**GET /exams/users/{user_id}/work/**

Получить находящуюся в работе экзаменационную сессию пользователя

**POST /exams/users/{user_id}/questions/ask/**

Задать вопрос пользователю. Находит актуальную экзаменационную сессию пользователя, достает из базы данных вопрос,
который еще не был задан пользователю в текущей экзаменационной сессии.

**GET /exams/users/{user_id}/questions/**

Находит все вопросы для актуальной экзаменационной сессии пользователя

**GET /exams/users/{user_id}/questions/unanswered/**

Находит последний вопрос, на который пользователь еще не дал ответа.

### Answers

**POST /answers/**

Дать ответ на вопрос

**GET /answers/exams/{exam_id}/**

Получить список всех ответов по экзаменационной сессии

**GET /answers/users/{user_id}/**

Получить список всех ответов пользователя

### Stats

**GET /stats/exams/{exam_id}/**

Получить статистику по экзаменационной сессии

**GET /stats/users/{user_id}/all/**

Получить всю статистику пользователя

**GET /stats/users/{user_id}/last/**

Получить статистику пользователя по последней экзаменационной сессии со списком ответов

## Сценарии использования

***

### Создание экзаменационной сессии и ее прохождение

1. Пользователь выбирает сколько вопросов ему необходимо в экзамене (от 1 до 10)
2. Посылается запрос на эндпойнт ```POST /exams/``` в теле запроса id пользователя и количество вопросов
3. После получения ответа со статус-кодом 200 сразу же шлется запрос на эндпойнт
   ```POST /exams/users/{user_id}/questions/ask/``` текст вопроса показывается пользователю
4. Пользователь отвечает на вопрос. Ответ пользователя отправляется на эндпойнт ```POST /answers/```
5. Шаги 3-4 повторяются пока при отправке запроса на эндпойнт ```POST /exams/users/{user_id}/questions/ask/``` не
   вылезет ошибка со статус-кодом 400 и следующим сообщением в теле ответа: *"Для пользователя с id = '1'
   экзаменационная сессия, находящаяся в работе, не найдена!"*
6. Отправляется запрос на ```GET /stats/users/{user_id}/last/``` для получения статистики. Статистика отображается
   пользователю. Экзаменационная сессия завершена, пользователь может начинать новую!

### Получение общей статистики пользователя

Отправить запрос на эндпойнт ```GET /stats/users/{user_id}/last/``` ответ показать пользователю

## Способ запуска приложения

***
*Предполагается, что все действия выполняются в директории ```backend```*

1. Создать виртуальное окружение
2. Установить все зависимости, выполнив команду:

```commandline
pip install -r requirements.txt
```

3. Создать файл .env по аналогии с файлом .env.example
4. Запустить postgresql в докере, выполнив команду

```commandline
docker compose -f docker/storage.yml -f docker/file_storage.yml --env-file .env up
```

5. Прогнать миграции:

```commandline
alembic upgrade head
```

6. Запустить сервер на 127.0.0.1:8000

```commandline
uvicorn --factory src.main:create_app
```
